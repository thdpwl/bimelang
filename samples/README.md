# 예시 파일 (업로드 테스트용)

에디터(`/editor`)의 두 가지 업로드 기능을 테스트할 수 있는 샘플입니다.

## CAD 가져오기(DXF)
| 파일 | 단위 | 레이어 | 내용 |
|------|------|--------|------|
| `plan.dxf` | mm (자동감지) | `WALL` `COLUMN` `SLAB` | 영문 레이어, 선/원/폴리라인 (10×8m) |
| `office.dxf` | m (자동감지) | `벽` `기둥` `바닥` `지붕` | 한글 레이어, 바닥+지붕 (12×9m) |

→ "CAD 가져오기(DXF)" 버튼으로 업로드하면 레이어→객체 매핑 모달이 뜹니다. (자동 추정값 그대로 "변환하여 불러오기")

> **벽 양면선 병합**: 도면에서 벽을 두께 있는 두 평행선으로 그린 경우, 모달의 "벽 양면선을 두께 벽 1개로 병합"(기본 켜짐) 옵션이 두 선을 **중심선 + 측정 두께를 가진 벽 1개**로 변환합니다.
> "벽 최대 두께(두 선 간격)"보다 멀리 떨어진 평행선은 마주보는 별개 벽으로 보고 병합하지 않습니다. 위 샘플은 벽이 단일 중심선이라 병합 대상이 없습니다.

> **참고**: 위 DXF 파일은 **AutoCAD 2025로 직접 생성·검증**한 정식 파일입니다(AutoCAD에서 그대로 열림).
> 초기 버전은 손으로 작성한 약식 DXF여서 웹 파서로는 읽혔지만 AutoCAD에서는 열리지 않아, 실제 CAD로 재생성했습니다.

## CAD에서 직접 확인 (DWG)
웹 변환 결과가 실제 도면과 일치하는지 AutoCAD·Revit 등에서 확인할 수 있도록 동일 도면의 DWG도 포함했습니다.
| 파일 | 포맷 |
|------|------|
| `plan.dwg` | AutoCAD 2018 DWG (AC1032) |
| `office.dwg` | AutoCAD 2018 DWG (AC1032) |

> 웹 에디터의 입력 데이터는 **DXF / JSON** 입니다. `.dwg`는 CAD 확인용 사본입니다.
> (DWG를 쓰려면 AutoCAD 등에서 `다른 이름으로 저장 → DXF`로 내보낸 뒤 업로드하세요.)

## 업로드(JSON)
| 파일 | 내용 |
|------|------|
| `house.json` | 벽·슬래브로 구성된 단층 주택 |
| `pavilion.json` | 기둥 4개 + 바닥/지붕 슬래브 (개방형 파빌리온) |

→ "업로드(JSON)" 버튼으로 불러옵니다.

## 다운로드 산출물
에디터에서 "다운로드(JSON)" 또는 "IFC 내보내기"로 저장하면 파일명이 `BIMelang_<프로젝트명>` 으로 생성됩니다.
