# CAD-TO-BIM
CAD에서 추출한 정보를 BIM 모델링으로 내보낼 수 있는 Add-in 기능

## 문제인식
BIM은 객체지향 3D 모델링이라고 생각하면됨. 그러나 BIM 설계를 위한 상용화된 프로그램 (revit, archicad 등)은 가격이 너무 비싸며, 사용할 수 있는 인력이 매우 저조함. 따라서 ai 기술을 활용해서 사람들이 쉽게 작동할 수 있는 프로그램이 필요함. 추가로 현재 건축사사무소가 주로 CAD형식의 도면 혹은 pdf 형식의 도면을 가지고 있으므로 해당 형식의 도면을 자동으로 BIM으로 전환, 변환할 수 있는 기술이 탑재된 프로그램이 절실히 필요함.

## 솔루션 방향
2D 도면을 자동으로 3D 모델링으로 변환하는 Add-in 프로그램을 만들고자 함.
cad 파일이나, pdf 파일을 업로드하면 자동으로 bim 모델링으로 변환할 수 있어야 함.
BIM은 객체 지향 모델링 프로그램으로 객체 별로 생성해야함. 즉 기둥, 벽, 지붕, 바닥 등을 객체별로 생성할 수 있어야함. 

## OUTPUT 데이터
생성된 데이터는 IFC 파일로 추출되어야함. 해당 파일은 BIM으로 내보내는 표준 데이터임. 추출된 데이터는 revit, archicad 등 상용화된 프로그램에서도 열릴 수 있음

## 배포 형식

## 진행 현황 (2026-07-01 기준)

### 완료
- **웹 에디터 MVP**: 랜딩 페이지 + 3D 에디터 구현, 벽/슬래브/기둥 객체 생성·편집 지원
- **CAD → BIM 자동 변환**: DXF 도면 업로드 시 객체별(벽·슬래브·기둥) BIM 요소로 자동 변환
  - CAD 벽 양면선(2줄)을 두께 있는 벽 1개로 자동 인식·병합
- **단면도·입면도로 높이 자동 산정**: 평면도와 함께 단면도/입면도(DXF)를 넣으면
  수평선을 층 레벨로 인식해 벽·기둥 높이와 지붕 높이를 자동 계산 (값은 변환 창·요소 속성에서 수정 가능)
- **바닥·지붕 자동 생성**: 산정한 층 레벨과 평면 외곽을 결합해 층별 바닥(슬래브)과
  최상단 지붕을 자동 생성 (기존에는 벽·기둥 위주로만 생성)
- **IFC 내보내기 (Revit 호환)**: 유효한 IFC4(STEP) 파일 생성. Revit·ArchiCAD 등에서 정상 임포트 확인
  - 요소를 실제 IFC 타입(`IfcWall`/`IfcSlab`/`IfcColumn`)으로 내보내고 압출 솔리드 형상을 연결
  - `IfcProject → Site → Building → Storey` 공간구조, 단위(mm), OwnerHistory 등 필수 구조 포함
  - 유일 GlobalId 생성, 한글 이름 STEP 인코딩 처리
- **웹 배포**: GitHub Pages 자동 배포 파이프라인 구성, 라이브 서비스 운영 (http://bimelang.kr/)
